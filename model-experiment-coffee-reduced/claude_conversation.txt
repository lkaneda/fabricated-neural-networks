  ---
  Proposed Process: Coffee Prototype Classifier

  Goal: 7-class classifier (americano, cappuccino, cortado, espresso, iced_coffee, latte, nitro_cold_brew) with no
  training — only one reference image per class.

  ---
  Step 1: Find a Source Model

  Search HuggingFace for a food/beverage classification model whose training classes include coffee drinks. The best
  candidate is nateraw/food (ViT-Base fine-tuned on Food-101), which includes classes like espresso, cappuccino, latte,
  coffee_cake, etc. — meaning the backbone already encodes coffee-specific visual features.

  Step 2: Extract Features (Prototype Vectors)

  Load the ViT-Base backbone frozen. For each of the 7 reference images in dataset/, run a forward pass and extract the
  768-dim CLS token embedding. These become the 7 prototype vectors.

  Step 3: Build the Classification Head

  Replace the source model's N-class head with a nn.Linear(768, 7) (no bias initially). Set each row of the weight matrix
   to its prototype vector. Then apply alpha-separation: for each pair of visually similar classes (e.g. latte vs.
  cappuccino), push their weight vectors apart along the discriminative direction by alpha=8.0.

  Step 4: Tune Biases Using the Test Set

  Run inference on all 70 test images. For each class, inspect the logit score distribution. Set biases to calibrate
  decision boundaries — shifting them so each class "wins" in its correct range.

  Step 5: Evaluate (Round 1)

  Run the full 70-image test set and print accuracy per class + confusion matrix.

  Step 6: Improve

  Based on the confusion matrix, identify the most-confused pairs and adjust:
  - Alpha-separation for those specific pairs
  - Bias adjustments
  - Possibly re-examine which prototype embeddings are most discriminative

  Step 7: Evaluate (Round 2)

  Re-run and report final accuracy.

  ---
  Output: A single coffee_model.bin file (state dict) + coffee_model.py (model class) that can classify coffee images.
