⏺ 2/3 is a solid result for a model with zero training — just 15 prototype images and no fine-tuning. And the one failure is the most understandable one: the model is doing nearest-neighbor in
   feature space, and the backbone's spatial features picked up on "small cup relative to scene" → cappuccino, which is exactly what a human would think at first glance too. It's not a model  
  failure so much as an optical illusion that fooled the feature extractor the same way it would fool a person.                                                                                 
                                                                                                                                                                                                
  The path to fixing that specific failure class would be training data with cups photographed at varying distances and with objects in the foreground — the model would need to learn that     
  absolute cup size doesn't determine drink type. That's not solvable with prototype vectors alone.

  - Hard ceiling without training: the 1-shot prototype approach is the best you can get from one image per class — more examples per class fed through the same pipeline would improve
